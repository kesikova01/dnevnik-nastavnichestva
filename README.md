# Дневник наставничества

Простой веб-дневник для участников программы наставничества: ежедневные записи (сон, вода, движение, благодарность), лист чувств/триггеров, задания по отношениям. Наставник видит все записи во вкладке «Наставнику» и может экспортировать их для разбора.

## Настройка перед первым запуском

Сайту нужна база данных Supabase, чтобы записи участников сохранялись и были видны наставнику.

1. Зарегистрируйтесь на [supabase.com](https://supabase.com) и создайте новый проект (бесплатный тариф).
2. В проекте откройте **SQL Editor → New query**, вставьте содержимое файла [`supabase_setup.sql`](supabase_setup.sql) и нажмите **Run**. Это создаст таблицу `diary_entries`.
3. В проекте откройте **Project Settings → Data API** и скопируйте:
   - **Project URL**
   - **anon public key**
4. Откройте файл [`config.js`](config.js) и вставьте туда эти два значения вместо заглушек:
   ```js
   window.SUPABASE_URL = 'https://ваш-проект.supabase.co';
   window.SUPABASE_ANON_KEY = 'ваш-anon-key';
   ```
5. Сохраните `config.js` и закоммитьте изменения — после этого сайт готов к публикации.

Ключ `anon` предназначен для использования на клиенте (не секретный), доступ к данным ограничен политиками RLS из `supabase_setup.sql`.

## Публикация

1. Создайте новый репозиторий на GitHub и запушьте эту папку.
2. В [Vercel](https://vercel.com) → **Add New Project** → импортируйте репозиторий.
3. Framework Preset — **Other** (это статический сайт, сборка не нужна).
4. Deploy — Vercel выдаст ссылку вида `https://ваш-проект.vercel.app`, её можно отправлять участникам.

## Файлы

- `index.html` — весь сайт (разметка, стили, логика).
- `config.js` — данные подключения к Supabase (URL и anon key).
- `supabase_setup.sql` — SQL-скрипт создания таблицы, выполняется один раз в Supabase.
- `vercel.json` — минимальная конфигурация для Vercel.
